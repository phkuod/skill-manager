Screen 2

Skill Detail — File Blocks

Each skill's documentation is broken into individual file blocks — one per file in the skill directory. Markdown files render as rich markdown; all other files display as syntax-labeled code blocks.

http://localhost:5173/skill/claude-api
Screenshot: 02-skill-detail.png
SKILL.md is always first. Subsequent files (LICENSE.txt, language examples, shared references) follow alphabetically. Each block shows the relative path and, for non-markdown files, a language badge.
📄

Markdown Rendering

.md files render with headings, tables, code blocks, and GFM support via ReactMarkdown.

💻

Code Blocks

All other files display verbatim in a monospace block with a language badge (python, typescript, go, etc.).

⚡

Large File Guard

Files over 500 KB show "File too large to preview". Binary files are silently excluded.

API

REST API Endpoints

The Express server exposes a JSON API. All endpoints are prefixed with /api.

Method Path Description
GET /api/health Health check — returns status and skill count
GET /api/skills List all skills (supports ?search= and ?category= filters)
GET /api/skills/:name Skill detail — metadata, content, and install paths
GET /api/skills/:name/files All files in the skill directory as [{ path, content, language }]
GET /api/skills/:name/zip Download the skill as a ZIP archive
Running the System

How to Start

Use manage.js from the project root for all operations.

🚀

Development

node manage.js dev
Starts Express API on :3001 and Vite on :5173.

🏗️

Production Build

node manage.js build
Builds React app to client/dist/, served by Express.

🧪

Tests

node manage.js test
Runs all 180 tests (72 server + 108 client) via Vitest.
